# 開発ガイド

memologの開発体制やテスト状況をまとめています。

## ディレクトリ構造

```
obsidian-memolog/
├── src/        # アプリケーションコード
│   ├── ui/        # UI層
│   ├── core/      # ロジック層
│   ├── fs/        # ファイルアクセス層
│   ├── types/     # 型定義
│   └── utils/     # ユーティリティ
├── test/          # テストコード
├── docs/          # ドキュメント
├── main.ts        # エントリーポイント
├── manifest.json  # プラグイン定義
├── SPEC.md        # 仕様書
└── TODO.md        # 実装計画
```

## 利用可能なnpmスクリプト

- `npm run dev` - 開発モード（ホットリロード）
- `npm run build` - プロダクションビルド
- `npm run test` - テスト実行
- `npm run test:coverage` - テストカバレッジ測定
- `npm run test:watch` - テストウォッチモード
- `npm run lint` - Lint実行
- `npm run lint:fix` - Lint自動修正
- `npm run format` - コードフォーマット
